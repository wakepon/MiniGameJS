# Update Docs

コードを読んでゲーム仕様書（Spec/*.md）を更新する:

1. **doc-updater** エージェントを使用して以下を実行:

2. 主要なソースコードを解析:
   - `src/lib/game/` - ゲームロジック・型定義・定数
   - `src/hooks/` - 状態管理・レイアウト計算
   - `src/components/` - UIコンポーネント・描画処理

3. 仕様書を更新:
   - 既存の `Spec/*.md` ファイルとコードを比較
   - コードを正（Source of Truth）として仕様書を更新
   - 新機能があれば新規ファイルを作成
   - 削除された機能があれば仕様書から削除

4. 更新ルール:
   - パラメータ調整対象の値（色のRGB値、パディング値など）は概要表現に留める
   - ゲーム構造に関わる固定値（グリッドサイズ、スロット数など）は記載する
   - 各ファイルは800行以下に収める
   - `Spec/README.md` の目次を更新する

5. **重要**: コードの変更は行わない（仕様書のみ更新）
