# Code Review

コミットされていない変更の包括的な品質レビュー:

1. 変更ファイルを取得: git diff --name-only HEAD

2. 各変更ファイルについて以下をチェック:

**コード品質 (HIGH):**
- 50行を超える関数
- 800行を超えるファイル
- 4階層を超えるネスト
- エラーハンドリングの欠如
- console.log 文
- TODO/FIXME コメント

**ベストプラクティス (MEDIUM):**
- Mutation パターン（代わりに immutable を使用）
- コード/コメント内の絵文字使用
- アクセシビリティの問題（a11y）

3. レポートを生成:
   - 重大度: HIGH, MEDIUM
   - ファイル場所と行番号
   - 問題の説明
   - 修正の提案

4. HIGH の問題が見つかった場合はコミットをブロック
