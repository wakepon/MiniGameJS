# Refactor Clean

テスト検証を伴う安全な不要コードの特定と削除:

1. 不要コード分析ツールを実行:
   - knip: 未使用の export とファイルを検出
   - depcheck: 未使用の依存関係を検出
   - ts-prune: 未使用の TypeScript export を検出

2. .reports/dead-code-analysis.md に包括的なレポートを生成

3. 発見事項を重大度で分類:
   - SAFE: テストファイル、未使用ユーティリティ
   - CAUTION: API ルート、コンポーネント
   - DANGER: 設定ファイル、メインエントリポイント

4. 安全な削除のみを提案

5. 各削除の前に:
   - フルテストスイートを実行
   - テストがパスすることを確認
   - 変更を適用
   - テストを再実行
   - テストが失敗したらロールバック

6. クリーンアップした項目のサマリーを表示

テストを先に実行せずにコードを削除しないでください！
