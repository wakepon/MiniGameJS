# テスト要件

## テストタイプ:
**Unit Tests** - 個別の関数、ユーティリティ、コンポーネント
Integration Tests や E2E Tests は行わない

## テスト駆動開発（TDD）

必須のワークフロー:
1. まずテストを書く（RED）
2. テストを実行 - 失敗するはず
3. 最小限の実装を書く（GREEN）
4. テストを実行 - 成功するはず
5. リファクタリング（IMPROVE）

## テスト失敗時のトラブルシューティング

1. **tdd-guide** agent を使用する
2. テストの独立性を確認する
3. テストではなく実装を修正する（テストが間違っている場合を除く）

## 注意事項
- 明示的に指示された場所のみテストを作成する(orchestrate tdd-featureの場合も含む)
- 高カバレッジを目指さない。重要なロジックにのみ焦点を当てる
- テストはシンプルかつ理解しやすく保つ
