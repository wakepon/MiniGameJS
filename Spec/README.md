## 仕様書の管理方針

- **コードが正（Source of Truth）**: 仕様書とコードにズレがある場合、コードを正として仕様書を更新する
- **数値の記載ルール**: パラメータ調整対象の値は記載せず、動作の概要のみを記載する
- **ファイルサイズ**: 800行を超えたらカテゴリ別に分割を検討する
- **更新履歴**: 各ファイルに更新履歴を記載する

## 更新方法

doc-updater エージェントを使用してコードから仕様書を生成・更新します。
